# 수정하고 저장하기 - 라이프 사이클

# git 상태 - 라이프 사이클

  • untracted - 추적되지 않음
  • unmodified - 수정되지 않음
  • modified - 수정됨
  • staged - 커밋 예정

추적되지 않음(untracked) -> 추적에 추가(add - staged) -> 히스토리 추가(commit - unmodified) -> 파일 수정(modified) -> 히스토리 추가 예정(add - staged) -> 히스토리 추가(commit - unmodified)

상태 확인

git status

# 간단하게 보기
git status -s

# 파일 무시하기

  • '.gitignore' 파일 이름으로 만들어 정의
  • 설정 또는 개발 환경 파일 등 추적을 원치 않는 파일, 폴더 등의 규칙 정의
# file name: .gitignore

# 확장자가 a 인 파일 무시
*.a

# 확장자가 a 인 파일 중 lib.a 는 무시하지 않음
!lib.a

# 현재(.gitignore 파일 위치) 디렉토리에 있는 'TODO' 파일만 무시(서브 디렉토리는 무시하지 않음).
/TODO

# build 디렉토리에 있는 모든 파일 무시
build/

# doc/notes.txt 파일은 무시하고 doc/server/arch.txt 파일은 무시하지 않음
doc/*.txt

# doc 디렉토리 아래 모든(서브 디렉토리 포함) pdf 파일 무시
doc/**/*.pdf
