# SEO

Search Engine Optimization - 검색 엔진 최적화

# 구글의 검색 기본 작동 방식

  • 크롤링: 구글 크롤러(Googlebot)가 신규나 업데이튼 페이지를 찾아 색인
  • 색인: 페이지를 분석하여 파악하여 저장
  • 검색결과 게재: 검색어 및 여러 요소(위치, 언어, 기기, 이전 검색어 등)로 관련성 높은 페이지를 색인에서 찾아 노출

# 게재 및 순위 개선하기

  • 빠르게 로드되는 모바일 친화적인 페이지를 만듭니다.
  • 페이지에 유용한 콘텐츠를 포함하고 최신 상태로 유지합니다.
  • 양질의 사용자 환경을 보장하는 데 도움이 되는 Google 웹마스터 가이드라인 (opens new window)을 따릅니다.
  • SEO 기본 가이드 (opens new window)의 도움말과 권장사항을 읽어 보시기 바랍니다.
  • 양질의 검색결과를 제공하기 위해 Google에서 품질평가자에게 제공하는 가이드라인 (opens new window)을 비롯한 자세한 정보는 여기 (opens new window)에서 찾아볼 수 있습니다.

# robots.txt 활용

  • 크롤링을 허용할 부분과 차단할 부분을 구분하여 작성
  • 루트 디렉터리에 robots.txt 파일로 확인
  • 구글 검색 결과에서 페이지를 숨기는 기능으로 사용하면 안됨. 비밀번호나 noindex (opens new window)등 다른 방법 사용
# 검색 엔진을 정의
User-agent: *
# 허용
Allow: /
# 차단
Disallow: /private/
# sitemap.xml 파일 지정
Sitemap: http://www.example.com/sitemap.xml

# 고유하고 정확한 페이지 제목 사용

  • <title> 요소는 사용자 및 검색엔진에 페이지의 주제를 알려줌
  • 구글 검색 결과 내용에 반영됨
  • 지나치게 길거나 관련 없는 키워드는 사용하지 않도록
  • <meta> 에 설명(description), 키워드(keywords) 작성
    • 사이트 모든 페이지에 있도록
    • 페이지 마다 고유한 설명을 작성
    • 페이지 관련 내용을 서술
  • open graph 작성
    • SEO 직접적 관련 X
    • 미리보기에 사용되는 형태로 사용
    • 모바일에서 앱 링크를 연결할때 사용
<!-- title 태그 -->
<!-- 메인 페이지 경우 -->
<title>사이트 이름 - 사이트 설명</title>
<!-- 서브 페이지 경우 -->
<title>페이지 제목(콘텐츠 제목) | 사이트 이름</title>

<!-- meta 태그 -->
<meta name="description" content="사이트/페이지 설명 작성">
<!--meta name="keywords" content="사이트/페이지 관련 키워드 ',' 로 구분하여 작성"-->

<!-- 오픈 그래프 -->
<!-- og:기본 -->
<meta property="og:title" content="사이트/페이지 타이틀">
<meta property="og:type" content="사이트/페이지 형태(website, video, audio, article, ..)">
<meta property="og:url" content="사이트/페이지 주소">
<meta property="og:image" content="사이트/페이지 대표 이미지 주소">
<!-- og:옵션 중 권장 -->
<meta property="og:description" content="사이트/페이지 설명">
<meta property="og:locale" content="사이트/페이지 언어">
<meta property="og:site_name" content="사이트/페이지 이름">
<!-- 트위터 카드 -->
<meta property="twitter:card" content="트위터 카드 타입(summary, player, app, ..)">
<meta property="twitter:url" content="사이트/페이지 주소">
<meta property="twitter:description" content="사이트/페이지 설명">
<meta property="twitter:image" content="사이트/페이지 대표 이미지 주소">
<!-- 모바일 앱 -->
<meta property="al:ios:url" content="iso 앱 url">
<meta property="al:ios:app_store_id" content="iso 앱스토어 아이디">
<meta property="al:ios:app_name" content="iso 앱 이름">
<meta property="al:android:url" content="안드로이드 앱 url">
<meta property="al:android:package" content="안드로이드 앱스토어 패키지">
<meta property="al:android:app_name" content="안드로이드 앱 이름">
<meta property="al:web:url" content="안드로이드 앱 url">

# 사이트 URL 구조 개선

이해하기 쉬운 URL은 ...

  • 콘텐츠 내용을 전달하기 좋아야 하고
  • 크롤러가 문서를 크롤링 하기 쉬워야 하고
  • 검색결과에도 노출되니까

# URL 구조를 위한 권장사항

  • url에 단어 사용
  • 단순 디렉토리 구조 만들기
  • 특정 문서에 도달하기 위한 한 가지 형태의 url 제공
    • 가능한 url 하나만 사용
    • 우선적인 url로 301 리디렉션
    • <link rel="canonical" href="원본url(절대주소 권장)"> 을 활용

# 콘텐츠 최적화

  • 우수한 품질의 콘텐츠와 서비스 제공
  • 보다 나은 앵커 텍스트 작성
  • 이미지 사용의 최적화 - img 태그에 alt 속성 활용
  • 제목 태그의 적절한 활용 - <h1> 부터 <h6> 까지
