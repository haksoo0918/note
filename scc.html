# Simple Coding Convention

웹 프론트엔드 개발에 최소한의 심플 코딩 컨벤션

# 공통

  • 인코딩은 UTF-8 을 기본으로 한다.

  • 들여쓰기는 2 공백으로 한다.

  • 빈 줄은 두 줄 이상 사용하지 않는다.

  • 주석의 셩격에 따라 목적을 명시한다.

    • 변수나 함수의 상세설명을 주석의 기본으로 하고 따로 목적을 명시하지 않는다.
    • 기본 기능을 제외하고 임시로 내용을 감출 때나, 삭제 예정의 목적 등으로 주석을 사용할 땐, '임시', '삭제 예정' 등의 목적을 주석 제일 앞에 명시한다.

  • 태그나 속성은 특별한 경우를 제외하고 소문자를 사용한다.

  • 속성값은 큰따옴표 " " 를 사용하여 감싼다.

    • data 속성으로 json 을 사용한다던 지의 특별한 상황은 예외로 한다.

  • head 와 body 바로 안쪽 태그는 들여쓰기하지 않는다.

  • 속성의 배치 순서는 필수속성(img 에 alt 등) > id > class > 기타 속성 > data 속성 > 속성에 값을 할당하지 않아도 되는 속성(input 에 require, readonly 등)

  • 단일 태그 끝에 마침 표시는 생략한다.

  • layout 에서 container 또는 wrap 역할을 담당하는 태그는 마침 태그 이후에 주석으로 역할을 적는다.

  • 공통 스타일은 'common.css' 로, 그 외는 화면이나 해당 목적에 맞는 이름을 사용한다.

  • 최상단에 파일의 이름이나 작성자, 적성일 등의 설명을 주석으로 표기한다.

  • 속성값을 따옴표로 감싸야 하는 경우에만 사용하고, 가능한 생략 한다.

  • 기본 따옴표는 작은따옴표를 ' ' 사용한다.

  • 속성은 한 줄에 하나씩 처리한다.

  • 되도록 class 셀렉터를 사용한다.

  • 태그 셀렉터는 최대한 배제한다.

  • class 나 id 의 기본 네이밍 스타일은 케밥 케이스(kebab-case)를 사용한다.

  • 오직 스크립트 만을 위한 셀렉터로써의 클래스는 js- 를 접두어로 사용한다.

  • 하나의 소스를 여럿이서 수정하며 진행할 땐, 추가나 수정되는 부분에 날짜와 수정내용, 작성자 등을 주석으로 표기한다.

  • CSS 파일 구성은 SMACSS 를 따른다.

  • 기본 태그 및 유틸리티 성격의 클래스 스타일을 정의할 땐 OOCSS 를 따른다.

  • Module 또는 Component 형태로 스타일을 정의할 땐 BEM 을 따른다.

  • 가능한 strict 모드를 기본으로 사용한다. ie9 을 맞춰야 할 경우에는 생략한다.

  • 글로벌 변수는 필요한 경우 외에는 사용하지 않는다.

  • 함수는 가능한 선언식 보다 표현식을 사용하여 정의한다.

  • 제어문이나 함수 등에서 블럭 중괄호는 같은 줄에서 시작한다.

  • 네이밍 스타일은 카멜 케이스(camelCase)를 사용한다.

  • Vender 등 특별한 경우를 제외하고 HTML 에 body 안 최하단에 요청 및 정의한다.

  • 문자열은 작은 따옴표 ' ' 를 사용하여 감싼다. (json 등 특별한 경우 제외)

  • 시멘틱을 최대한 고려한다.

  • 스타일을 위한 id 를 사용하지 않는다.

  • 네이밍은 케밥 케이스(kebab-case)를 사용한다.

  • 약속어를 미리 정하고 사용한다.

  • 형태, 목적(의미), 상태 순으로 사용한다.

  • 네이밍 스타일은 스네이크 케이스(snake_case)를 사용한다.

  • 공통으로 사용되는 파일은 'common' 이란 이름을 사용한다.

  • 서비스 내용을 파일 이름으로 사용한다.

  • 네이밍 스타일은 프로젝트별 상황에 맞게 사용하거나 기본적으로 케밥 케이스(kebab-case)를 사용한다.
