# viewport meta tag

모바일 브라우저들은 뷰포트(viewport)로 알려진 가상 'window' 상에 페이지를 렌더링.
개발자들이 뷰포트의 크기와 스케일을 조정.

모바일에서 픽셀은 픽셀이 아님.
실제 해상도를 사용하지 않고 뷰포트에서의 정의를 따름.

기본 비율은 디스플레이의 밀도에 따라 달라짐.
200dpi 이하의 밀도에서는 1.0, 200~300dpi 에서는 1.5, 300dpi 이상 소수점 없는 정수가 됨.

# syntax

<meta name="viewport" content="width=device-width, initial-scale=1.0">
1

# attributes

  • width
  • height
  • initial-scale
  • maximum-scale
  • minimum-scale
  • user-scalable

# Canonical

중복/동일 콘텐츠 노출 방지 또는 통합 목적

  • 동일한 콘텐츠가 다양한 형태로 노출 되는 경우
  • 메인/표준의 형태/서비스를 정의
  • 검색 결과에 아름답게 반영

# 경우

  • 동일한 콘텐츠를 다양한 플랫폼에 노출할때
    • 메인 홈페이지, 운영하는 블로그, SNS, 다른 홈페이지 등 다양한 채널을 사용할때
    • 데스크탑 페이지, 모바일 페이지, amp 페이지 등 여러 url 을 사용할때

# 적용

  • 메인 콘텐츠를 제공할 곳을 정해서 그곳의 절대주소 url 을 적용한다.
<link rel="canonical" href="https://www.example.com/content-url/">

// webp.js
function WebpIsSupported(callback){
  // If the browser doesn't has the method createImageBitmap, you can't display webp format
  if(!window.createImageBitmap){
    callback(false);
    return;
  }

  // Base64 representation of a white point image
  var webpdata = 'data:image/webp;base64,UklGRiQAAABXRUJQVlA4IBgAAAAwAQCdASoCAAEAAQAcJaQAA3AA/v3AgAA=';

  // Retrieve the Image in Blob Format
  fetch(webpdata).then(function(response){
    return response.blob();
  }).then(function(blob){
    // If the createImageBitmap method succeeds, return true, otherwise false
    createImageBitmap(blob).then(function(){
      callback(true);
    }, function(){
      callback(false);
    });
  });
}

window.onload = () => {
  WebpIsSupported((isSupportWebP) => {
    if (!isSupportWebP) {
      const container = document.querySelector('.container'); // 개인적으로 body 가 나을듯.
      container.classList.remove('support-webp');
    }
  });
}

# min, max, clamp

상대 길이 단위 값을 활용하여 min 은 값중 작은 값을, max 는 값중 큰 값을, clamp 는 최소값에서 최대값 그리고 그 사이값 까지 정의하여 사용 가능하다

width: min(50%, 500px); /* 50% 와 500px 중 작은 값을 사용한다 */
height: max(50vh, 400px); /* 50vh 와 400px 중 큰 값을 사용한다 */
margin: clamp(30px, 50% + 10px, 100px); /* 최소 30px 최대 100px 그 사이 50% + 10px 의 값을 사용한다 */
1
2
3

상대 길이 단위들이 있다. %, em, rem, vw, vh 등 다양하다.

  • %는 부모 엘리먼트의 상대값이다.
  • em은 부모 엘리먼트의 폰트크기 비례값이다.
  • rem은 최상위(root) 요소 즉 html 의 폰트크기 비례값이다.
  • vw, vh에서 v는 viewport 를 가르킨다. 각 viewport 의 width 값과 height 값의 1% 값에 비례한다.
  • vmin, vmax는 viewport 에서 작은 값과 큰 값의 1%에 비례한다.
